# SEO Fundamentals — Design Spec

**Date:** 2026-04-06
**Status:** Approved
**Scope:** Approach B — Full Technical SEO

## Context

The game launched on April 1, 2026 and has ~875 visits in its first week, 100% from referral traffic (Reddit 59%, Facebook, Instagram). Zero organic search traffic. The site is missing basic SEO infrastructure — no OG tags, no sitemap, no structured data, generic meta descriptions shared across all pages.

**Goal:** Get the site properly indexed by Google, enable rich social previews on every share, and lay the foundation for organic search traffic.

**Canonical base URL:** `https://dewanggogte.com/games/bugs`
(not `whats-that-bug.vercel.app` — share links and primary traffic flow through the personal site)

---

## 1. Layout Changes (`src/layouts/Base.astro`)

### New Props Interface

```typescript
interface Props {
  title: string;
  description: string;
  ogImage?: string;       // defaults to "/og-default.png"
  ogType?: string;        // defaults to "website"
  canonicalPath?: string;  // e.g. "/" or "/play" or "/leaderboard"
}
```

### New `<head>` Tags

Add after existing meta tags:

- `<link rel="canonical" href="{CANONICAL_BASE + canonicalPath}">` 
- `<meta property="og:title" content="{title}">`
- `<meta property="og:description" content="{description}">`
- `<meta property="og:image" content="{CANONICAL_BASE + ogImage}">`
- `<meta property="og:url" content="{CANONICAL_BASE + canonicalPath}">`
- `<meta property="og:type" content="{ogType}">`
- `<meta property="og:site_name" content="What's That Bug?">`
- `<meta name="twitter:card" content="summary_large_image">`
- `<meta name="twitter:title" content="{title}">`
- `<meta name="twitter:description" content="{description}">`
- `<meta name="twitter:image" content="{CANONICAL_BASE + ogImage}">`

### Semantic HTML

Wrap `<slot />` in `<main>`:

```html
<main>
  <slot />
</main>
```

Replace the hardcoded generic `<meta name="description">` with the per-page `description` prop.

- Dark background matching the game's theme (`--bg` color)
- Beetle emoji or stylized bug icon
- "What's That Bug?" in large text
- Tagline: "Can you identify 1,000+ insects?"

Save to `public/og-default.png`.

No iNaturalist photos used — avoids CC BY attribution requirements on the share card.

## 4. Technical SEO Infrastructure

### astro.config.mjs

```javascript
import { defineConfig } from 'astro/config';
import sitemap from '@astrojs/sitemap';

export default defineConfig({
  site: 'https://dewanggogte.com',
  base: '/games/bugs',
  integrations: [sitemap()],
  build: {
    assets: '_wtb',
  },
});
```

Note: Adding `base` may affect existing path logic in the inline scripts (Base.astro lines 29-47 handle subpath detection at runtime). Need to verify compatibility — Astro's `base` config rewrites asset paths at build time, while the existing `__BASE` detection works at runtime. If they conflict, omit `base` from config and keep the runtime detection, just set `site` only.

### robots.txt

Create `public/robots.txt`:

```
User-agent: *
Allow: /

Sitemap: https://dewanggogte.com/games/bugs/sitemap-index.xml
```

### sitemap.xml

Auto-generated by `@astrojs/sitemap` at build time. Will include:
- `/`
- `/play`
- `/leaderboard`

### manifest.json

Create `public/manifest.json`:

```json
{
  "name": "What's That Bug?",
  "short_name": "What's That Bug?",
  "description": "A free guess-the-bug game with 1,000+ species.",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#1a1a1a",
  "theme_color": "#c97a4a",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

Add to `<head>` in Base.astro:
- `<link rel="manifest" href="/manifest.json">`
- `<link rel="apple-touch-icon" href="/icon-192.png">`
- `<meta name="theme-color" content="#c97a4a">`

### Favicon PNGs

Generate 192x192 and 512x512 PNG versions of the beetle emoji for the manifest and apple-touch-icon. These go in `public/`.

## 6. Risks & Tradeoffs

- **Canonical URL locks to `dewanggogte.com/games/bugs`**: If the game ever moves to its own domain, canonical URLs need updating. Acceptable for now.
- **`@astrojs/sitemap` dependency**: Official Astro package, well-maintained, lightweight. Low risk.
- **`base` config may conflict with runtime `__BASE` detection**: The existing inline script in Base.astro detects subpath at runtime. Astro's `base` config rewrites paths at build time. Need to test compatibility — if they conflict, use `site` only and omit `base`.
- **Play page SEO is limited**: Content is JS-rendered, so Google's JS renderer must execute it. The index page is where organic traffic will land; play page SEO matters less.
- **No per-set OG images**: All pages share the same default card. Per-set differentiation is deferred to Approach C.
- **OG image won't show bug photos**: Text-only card is less eye-catching than a photo card, but avoids CC BY attribution complexity at this stage.
