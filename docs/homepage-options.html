Current homepage

It is tidy, but undersells the promise.

The page gives users a route into the game, but it reads more like a menu than a game landing screen. The strongest concepts should be visible before any click: guessing from real photos, a daily mystery, modes, and what players learn.

Source: src/pages/index.astro
High impact

Modes are hidden behind the set cards

Users see "Bugs 101" and "All Bugs", but not Classic, Time Trial, or Streaks until after a click. That makes the first action feel like configuration instead of play.

Bug

Player stats card likely never inserts

The script inserts stats before .mode-group, but the homepage now uses .homepage-section, .play-cards, and .themed-buttons. Returning players lose a useful retention hook.

Layout

Six themed sets sit in a five-column grid

The desktop CSS uses repeat(5, 1fr) for six cards, creating a 5 + 1 layout. A 3 x 2 grid would look intentional and scan better.

Positioning

Daily Challenge is too secondary

The banner is useful, but it is visually a strip below the title. It could be the homepage's most habit-forming CTA and should feel like a feature.

Copy

The hero is factual, not motivating

"1,000+ species. How many can you name?" is clear, but it does not show the user what they will do in the first 10 seconds.

Content

The learning section sounds generic

The paragraph says the dataset is broad and verified. Better: show the kinds of tells players learn, such as wing shape, body segments, antennae, and habitat clues.

When to choose this

Use this if analytics or interviews show that new users are unsure what "Bugs 101" vs "All Bugs" means, or if the Daily/Time Trial/Streak modes are more important than set selection.

  • Pros: fewer abstract labels, stronger first-click clarity, easier to promote Daily and Multiplayer.
  • Cons: changes the mental model and may require rethinking the mode picker.
  • Risk: expert users who know exactly what set they want may need one extra click unless set shortcuts remain visible.

Homepage changes

  1. Replace the small centered hero with a stronger headline, short explanation, and one visual photo card.
  2. Promote Daily Challenge from strip to feature card with the current solved/missed/streak state.
  3. Add mode preview rows to Bugs 101 and All Bugs, or make the cards link directly to a default mode while keeping the mode picker as secondary.
  4. Set desktop themed cards to 3 columns so six sets form a balanced 3 x 2 grid.
  5. Replace the long "What You'll Learn" paragraph with three cards for body plan, field marks, and genus memory.

Small code fixes noticed

  1. Change the stats insertion target from .mode-group to the first .homepage-section, or place it explicitly after the daily card.
  2. Audit the old homepage CSS blocks for .mode-group and .mode-btn; they appear to be left over from the previous homepage layout.
  3. Keep the overlay accessible if mode previews become direct links: keyboard focus, Escape close, and returning focus to the trigger still matter.
  4. Use one or two real observation photos in the hero only if loading performance stays acceptable. Otherwise use a CSS/photo placeholder.
  5. After implementation, check mobile at 360px, 390px, and 768px widths because the homepage currently packs several dense CTA grids.
